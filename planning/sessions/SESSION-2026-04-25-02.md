## Bugs / issues discovered

One self-inflicted, caught + fixed mid-stream:

- **SQLAlchemy `before_update` + `get_history` silently accepts illegal transitions after a commit.** The initial Task 1 implementation used `@event.listens_for(Tool, "before_update")` + `get_history(target, "lifecycle_state")`. After a commit, SQLAlchemy expires attributes; the next setattr + flush has `history.deleted = []` because the old value never loaded. The listener's `if old is None: return` short-circuit silently accepted transitions. Caught by the first run of `TestEventListener::test_direct_setattr_illegal_transition_raises`. Fix: switched to the `set` attribute event with `active_history=True`, which fires immediately on setattr and forces the committed value to load from the DB. Also documents the pattern in a regression test (`test_listener_catches_expired_attribute_after_commit`) so future sessions see the trap.
