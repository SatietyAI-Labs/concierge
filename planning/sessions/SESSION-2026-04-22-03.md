### Methodological pin — validator scope is SHAPE, not CONTENT

The real call returned **csvkit** (a pack, `is_in_catalog=True`, slug `csvkit`) as rank 1. The moneyshot baseline (DECISIONS `[2026-04-22 16:27]`) returned **csvstat** (a tool *inside* the csvkit pack) as rank 1. Different content at the slug level; validator correctly flagged neither.

This is the boundary that must not erode: **shape is testable, content is stochastic.**

- **Validator scope (shape, CI-testable):** rank-sequence density, `tool_slug`/`is_in_catalog` consistency, stop-reason + effort in known sets, gap-report sub-section firing rules, SNA wording presence (the behavioral-voice invariants from CLAUDE_CODE_GAP_PREAMBLE), pinned markdown grammar.
- **Out of validator scope (content, soak-signal only):** which specific tool wins rank 1, which alternatives appear, exact slug strings, ranking order between tools, confidence distribution across the list, natural-language rationale wording. Opus 4.7 at `effort=xhigh` with no `temperature` parameter is stochastic by design — what looks like "drift" at the content level is actually working-as-intended variance under a pinned model.

Today's csvkit-vs-csvstat divergence is benign content variance, not shape drift. The validator correctly stayed quiet because the shape invariants held on both responses. This is the correction-in-reframing that `[2026-04-21 18:00]`'s DECISIONS correction note anticipated when it shifted the variance-source mechanism from temperature-pinning to effort-pinning: pinned effort narrows but does not eliminate variance at the content level. That's a feature of Opus 4.7, not a bug we should attempt to test around.

**Durable rule for future contributors:** assert on shape invariants in CI (validator, `test_recommend_pipeline.py`, `test_meta_tools_gap_report.py`). Capture content observations in soak-log analysis and DECISIONS entries as drift-comparison signals. Never assert in CI on specific tool names, ranking order, or exact rationale strings — that path leads back to the anti-pattern the operational-first pivot warned about: over-specifying reality into tests that succeed only under conditions a stochastic model happens to reproduce.
