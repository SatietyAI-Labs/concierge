## Bugs / issues discovered

**Code bugs caught and fixed during the build arc:**

- **Regex newline-crossing in `_FIELD_RE`** (caught by
  `test_parse_sections_empty_fields`). The pattern used `\s*` between
  the field's closing `**` and the value capture group; `\s` includes
  `\n`, so empty values consumed the newline and matched content from
  the next line. Fix: `[ \t]*` stays within the current line.
  Introduced in N3, fixed in N3.

- **`autoflush=False` in `get_session_factory`** (caught during N3
  live smoke; hidden by default-autoflush in test fixtures). The
  insertâ†’queryâ†’update upsert pattern needs autoflush=True so the query
  sees the pending insert; with False, the query returned None for the
  just-added row, a second insert ran, and commit hit the UNIQUE
  constraint. Fix: `autoflush=True` (SQLAlchemy default). Added
  `test_ingest_via_production_session_factory_on_disk` as a regression
  guard. Introduced in N2, fixed in N3.

- **SQLite `:memory:` + TestClient thread divergence** (caught when
  the first API test ran â€” "no such table: tools"). SQLite default
  creates a new in-memory DB per connection; TestClient dispatches
  requests on a separate thread, so `create_all` and the request query
  saw different :memory: instances. Fix: `poolclass=StaticPool` pins to
  a single connection. Introduced latent in conftest from N2-era
  fixtures, surfaced in N4, fixed in N4.
