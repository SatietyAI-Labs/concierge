## Bugs / issues discovered

**Caught and fixed during N8 implementation (not shipped broken, no regression risk):**

- **Fixture files planted post-service-init are DB-invisible.** First-pass N8 roundtrip test copied a fixture `.md` into the tmp `pending/` folder and expected `GET /requests/pending` to return it — but the reconciliation pass runs once at service init via lifespan, and files added later have no DB row. Fix: helper `_plant_and_reconcile(root, db_session, src, filename)` that copies the file + runs `reconcile_lifecycle(...)` explicitly. Mirrors how a Day-5 soak operator would need to trigger a manual reconcile if they suspect drift — surfaced the "manual reconcile endpoint" as a Day-5 candidate.
- **`/health` counter bump invisible when fixture service constructed with own counter.** The N7 counters-singleton pattern (service's `__post_init__` defaults to `get_counters()`) means service ops bump the module singleton that `/health` reads. The N8 test fixture was constructing `LifecycleCounters()` directly and the `/health` read saw the separate module singleton — zero increments. Fix: use module singleton via `reset_counters_for_tests()` + let service default. Worth noting in case future tests want isolated counters per test — we'd need a way to inject the counter source into `/health`.

**Caught and fixed during N10 implementation:**

- **Print-lint false-positive on docstring mentions.** First-pass `test_shim_print_lint.py` was a grep-style scan; it flagged docstring paragraphs that *mentioned* `print(` as a term (e.g. "there are no `print()` calls anywhere"). Fix: AST-walk for actual `ast.Call` nodes with `func.id == "print"`; docstrings become string literals that ast.walk skips at the Call level. Same AST approach used for the `sys.stdout.*` attribute check. Caught entirely within the test suite — no shim-runtime behavior affected.

**Non-issues worth noting for soak-phase attention:**

- `test_export.py` + `test_ingest.py` legacy-corpus tests were silently skipping under the new isolated `lifecycle_root` default; redirected to read `_legacy/tool-requests/` directly. If the legacy symlinks are ever broken, those two tests go back to skipping gracefully — not a failure, just a documented skip.
