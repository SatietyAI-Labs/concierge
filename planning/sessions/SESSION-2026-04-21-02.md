### N5 execution

N5 shipped reflecting the pivot.

- `core/memory.py` — `MemoryClient` class, `MemoryHit` frozen dataclass,
  `MemoryUnavailableError`, `make_memory_client()`/`get_memory_client()`
  factories.
- Default memory dir: `~/.concierge-memory/` (isolated per pivot).
- Wire-compat with moltbot on storage: same collections (`memories`,
  `identity`), same embedding model (`all-MiniLM-L6-v2`), same tag
  serialization (JSON-encoded list in metadata), same `mem_` id
  prefix, same naive-UTC ISO timestamp format.
- One documented divergence: tag-filter semantics. Modern ChromaDB
  (1.5.8) dropped `$contains` metadata operator (document-only now);
  N5 moves tag filtering to post-hoc Python with exact-membership
  semantics (stricter than moltbot's substring match; over-fetches
  5× to maintain result count). Wire-compat preserved on storage;
  only filter semantics differ.
- Lazy init: `import core.memory` is cheap; ChromaDB client and
  `sentence-transformers` only load on first operation.
- Graceful degradation: `MemoryUnavailableError` narrow exception
  raised on any backing-store failure (ChromaDB init, embedding-
  model load, query exception). N6 is expected to catch this and
  serve a recommendation without memory context.
- Not implemented (deliberate scope trim): `delete`, `update`,
  `identity_get`, `identity_set`, async variants. No consumer yet;
  add when N7 / N19 / future feature materializes a call site.
- Heavy install: `chromadb>=0.5.0` + `sentence-transformers>=3.0.0`
  added to main deps (not optional — memory is operational core
  per pivot); torch pulled ~2GB into venv. One-time cost.
- Tests: 32 new (26 fast + 6 integration). Pytest marker
  `integration` added so `pytest -m "not integration"` skips heavy
  roundtrip tests.

## Bugs / issues discovered

**Caught and fixed during N5 implementation (not shipped, no
regression risk):**

- **ChromaDB 1.x `$contains` metadata operator removal.** Moltbot's
  `{"tags": {"$contains": ...}}` filter pattern silently returns
  empty in chromadb 1.5.8 (the operator is now document-content
  only, not metadata). First-attempt N5 inherited moltbot's pattern
  → 2 integration tests failed. Fix: moved tag filtering to
  post-hoc Python with exact-membership semantics; over-fetched 5×
  to maintain result counts. Documented as intentional divergence
  from moltbot in `core/memory.py` header and in the commit message.
- **`datetime.utcnow()` deprecation warning.** Replaced with
  `datetime.now(timezone.utc).replace(tzinfo=None).isoformat()` to
  produce the same naive-UTC ISO format moltbot stores (preserving
  wire-compat on a shared store) without the deprecation noise.
